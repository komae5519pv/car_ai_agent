# IDOM Car AI — デモシナリオ

> **対象:** IDOM 経営層・CIO・営業本部長
> **目的:** 「AIが営業を変える」だけでなく、「AIを継続的に改善できる仕組み」まで見せて Wow を引き出す

---

## デモで伝えたいこと（1枚サマリー）

| 現状の課題 | このデモで見えること |
|-----------|-------------------|
| 営業担当が顧客の「本当のニーズ」を掴めないまま提案 | AIが商談録音から深層ニーズを自動抽出し、ベスト3台を即座に提示 |
| 担当者によって提案品質にバラつき | AIが全担当者の提案を均質化・底上げ |
| AIが間違えても誰も気づかない | LLM Judgeが自動で品質スコアを付け、異常を即検出 |
| 問題を発見しても何が原因かわからない | トレースで実行ステップを可視化し、根本原因を特定できる |

**Databricksだから実現できる理由:** 顧客データ・在庫データ・商談録音・AI推論ログ・評価データが Unity Catalog に統合されているため、「データ → AI → 改善サイクル」が1つのプラットフォームで回る。

## Scene 1: 山田優子さんの担当になった営業担当者の朝

### 背景（デモ前に語るストーリー）

> 山田優子さん（38歳）が来店。夫、小学生の子供2人、72歳の義母と同居。
> 「義母も乗りやすい車がいい」「アルファードいいなと思うけど予算が...」
> 商談担当者はこの録音を聞き返す時間もなく、次の接客へ。

**Before（現状）:** 担当者の頭の中だけに顧客情報がある。引き継ぎは口頭。提案はカンと経験頼み。

### デモ操作（営業画面）

**① 顧客リストから山田優子さんを選択**
- 画面左: 顧客一覧。検索・フィルタで即到達。

**② インサイトパネルを確認**（自動生成）

```
深層ニーズ（AI抽出）:
  ✓ 義母の乗り降りのしやすさ（低床設計が必須）
  ✓ 子供2人の習い事送迎に使いやすい配置
  ✓ 「大きい車は不安」→ 運転しやすいサイズ感

購買意欲: 高（「車を探してる」と明言）
核心インサイト: アルファードへの憧れはあるが現実的。
              「義母への気遣い」が本当の決め手になる。
```

**→ Wow ポイント①:** 「担当者が30分かけて読む商談録音を、AIが3秒で要約して本質を抽出している」

**③ おすすめ車両パネルを確認**

```
第1推薦: トヨタ シエンタ（マッチ度 95%）
  → 義母の乗り降りに最適な低床設計
  → 予算内（188万円〜）
  → 「義母も乗りやすいだろうし」の発言を根拠として引用

第2推薦: ホンダ フリード（92%）
  → Honda SENSING で安全性◎
  → 3列シートで急な送迎にも対応
```

**→ Wow ポイント②:** 「推薦理由が『マニュアル通り』ではなく、この顧客の言葉を根拠にしている」

**④ Ask AI サイドバーで深掘り**

> 「なぜシエンタが第1推薦なの？他の選択肢は？」

AIが即座に回答。担当者は顧客の前でリアルタイムに確認できる。

**→ Wow ポイント③:** 「営業担当者がAIに質問しながら商談できる。まるにベテランが隣にいる感覚」

## Scene 2: AIが間違えた。でもちゃんと気づいて直せる

### 背景（デモ前に語るストーリー）

> 松本あかね（26歳・独身OL）への推薦で、AIが予算220万円を無視してハリアー（330万円〜）を推薦してしまった。
> これをAIシステムが自動で検出し、管理者にアラートを送った。

**Before（現状）:** AIが間違えても誰も気づかない。クレームが来て初めてわかる。

### デモ操作（管理者画面）

**① ダッシュボードを開く**

画面上部に赤いバナー:

```
⚠️ 要対応: 1件の品質アラート
LLM Judgeが低スコア（2.5以下）の推論を検出しました
                                    [ 確認する → ]
```

**→ Wow ポイント④:** 「AIが自分の失敗を自動で検出してアラートを出している。人間のレビュー待ちゼロ」

**② 品質モニタリング → 要対応タブ**

松本あかねの推論ログが1件表示:

```
[要対応]  車両推薦  松本あかね  42分前

質問: 松本あかね（26歳・独身OL）予算150-220万円の最適車両を推薦してください
回答: ハリアー(91%)、レクサスRX(88%)を推薦

Relevance 4.0  Faithfulness 1.5  Helpfulness 1.5  → Overall 2.3 / 5
```

**③ ログを展開 → Judge判定理由**

```
【問題検出】ハリアー（332〜438万円）とレクサスRX（550万円〜）は
いずれも顧客の予算上限（220万円）を大幅に超えています。

【原因分析】retrieve_customer_dataスパンでは budget_max=2,200,000 が
正しく取得されていますが、search_vehicle_inventoryスパンで予算フィルターが
適用されておらず、全車両が候補に含まれています。

【推奨アクション】検索クエリに
「price BETWEEN budget_min AND budget_max」フィルターを追加してください。
```

**④ 実行ステップ（Trace Timeline）を確認**

```
✓  retrieve_customer_data         198ms
   budget_max: 2,200,000 ← 正しく取得されている

⚠️  search_vehicle_inventory       191ms
   filter: price > 0  ← 予算フィルター未適用！
   ⚠ 予算フィルター未適用: budget_max=2,200,000 が検索クエリに反映されていません

✓  ChatCompletion                 1,756ms
   ⚠ 推薦車両が顧客の予算上限を超過しています
```

**→ Wow ポイント⑤:** 「問題がどのステップで起きたか、コードを読まなくても一目でわかる」

**⑤ 人間評価を追加**（以下の内容を入力）

| 項目 | 入力内容 |
|------|---------|
| 総合評価 | ★☆☆☆☆（1/5） |
| フィードバック | 予算220万円の顧客にハリアー（330万円）を推薦するのはあり得ない。顧客が予算オーバーとわかった瞬間に信頼を失う。予算フィルターを必ず適用すること。 |
| 期待される正解 | 予算150-220万円内の車両（ノート、フィット、シエンタ等）を推薦し、それぞれの特徴と顧客ニーズへのマッチ理由を説明する。 |

**→ このフィードバックが Ground Truth データになり、次の mlflow.evaluate() での再評価に使われる。**

**→ Wow ポイント⑥:** 「営業マネージャーの判断がそのままAIのトレーニングデータになる仕組みがある」

## Scene 3: なぜ Databricks じゃないとできないのか

### 統合データ基盤（Unity Catalog）

```
顧客マスタ         → customers テーブル
在庫車両データ      → vehicle_inventory テーブル
商談録音テキスト    → customer_interactions テーブル
AIインサイトキャッシュ → customer_insights テーブル
AI推薦結果キャッシュ → recommendations テーブル
車両画像           → images Volume
RAGナレッジ文書     → knowledge Volume
```

**全データが1つのカタログに統合。** 部門をまたいだデータ活用もガバナンスを保ちながら実現。

### MLflow Tracing（完全な可観測性）

```python
@mlflow.trace
def recommend_vehicles(customer_id: str):
    customer = retrieve_customer_data(customer_id)   # ← 各ステップを自動記録
    candidates = search_vehicle_inventory(customer)
    return llm_recommend(customer, candidates)
```

デコレータを1行追加するだけで、全ステップの入出力・レイテンシ・トークン数が自動記録される。

### Foundation Model API

```python
# モデルをAPI1本で切り替え可能
LLM_MODEL = "databricks-claude-sonnet-4"
# → "databricks-llama-3-3-70b" に変更するだけでモデルチェンジ完了
```

**コスト最適化やモデルアップグレードがゼロコード変更で実現。**

### mlflow.evaluate()（LLM-as-Judge の自動化）

```python
# 定期バッチで自動評価を実行
results = mlflow.evaluate(
    data=inference_table_df,
    model_type="question-answering",
    evaluators="default",
    extra_metrics=[budget_compliance_metric]  # 独自メトリクスも追加可能
)
```

**人間が見切れない量のログを、AIが自動で品質チェック。問題だけ人間に届ける。**
